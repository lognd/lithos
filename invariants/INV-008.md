Checked mirror of INV-8 (Target additivity) in
`docs/spec/regolith/13-invariants.md`; that file is the normative
source.

Mechanism: `check_target_reserves` rejects a target build whose draws
exceed a declared reserve; targets cannot modify base contracts
(syntactic), and base realization is reused verbatim when unperturbed.

Argument summary: if the base snapshot byte-matches, its evidence keys
match (INV-1); if a target perturbs it, keys differ and
re-verification is forced -- reuse is never silent.

Anchor (primary enforcing site): `crates/regolith-ir/src/system.rs :: check_target_reserves`
