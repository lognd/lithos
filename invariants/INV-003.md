Checked mirror of INV-3 (Hint droppability) in
`docs/spec/regolith/13-invariants.md`; that file is the normative
source.

Mechanism: `Obligation.hints` carries discharge hints from the
claim/symmetry pass as inert data -- the model API gives them no role
in domain validity or coverage claims; anything load-bearing must
arrive as a checked fact, a registry record, or an `assume!`, none of
which the hint channel can carry.

Argument summary: hints are structurally guidance-only data riding
alongside (never inside) the fields the discharge model reads for
validity, so stripping them cannot change a verdict, only search cost.
