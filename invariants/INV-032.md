Checked mirror of INV-32 (Tap-map/artifact agreement) in
`docs/spec/regolith/13-invariants.md`; that file is the normative
source.

Mechanism: `check_tap_agreement` compares the map's allocated
`(channel, target_path)` rows against every `REGOLITH-TAP ch=<n>
target=<path>` marker re-parsed out of every emitted file (board
`tap_placements.json`, firmware `debug_taps.h`, HDL `debug_taps.v`).
Either uncovered difference is a `tap_map_artifact_mismatch`
diagnostic and `ship` returns `Err` before the manifest is written.

Argument summary: the two sides of the comparison are independently
derived (derivation-time tap set vs. bytes actually written), so
agreement cannot be assumed by construction and must be, and is,
checked; a release-profile ship never derives a tap set at all, so the
check is vacuously satisfied there (byte-identity pinned by golden
equality).

Anchor (primary enforcing site): `python/regolith/backends/debug_taps.py :: check_tap_agreement`
