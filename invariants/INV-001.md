INV-1 (Evidence binding); that file remains the normative source --
this file exists so `frob check` can gate on it.

Mechanism: obligations are content-addressed over (claim, subject
snapshot, givens, registry record hashes, model-registry version) via
`Obligation::content_hash`/`evidence_cache_key`. Any semantic input to
a verdict is part of the key, so no source edit can re-label existing
evidence -- editing boundary truth, materials, or contracts produces a
*different* obligation with no evidence yet.

Argument summary: content addressing over every semantic input is a
sound proxy for "same obligation, same key" because there is no
back-channel into the hash that a source edit could miss.
