Checked mirror of INV-28 (Evidence attribution) in
`docs/spec/regolith/13-invariants.md`; that file is the normative
source.

Mechanism: `verify_attestation` is a total three-valued function --
`Valid(tier)` against a designated magnetite key, `Unsigned` (the
`community` floor) when absent, or `Invalid(reason)` when present but
unverifiable; the release gate refuses any discharged result whose
claim floor exceeds its conferred tier.

Argument summary: the signature binds to the content address, whose
collision resistance is blake3's, so it cannot be transferred to other
evidence; both non-`Valid` arms map to explicit evidence states, so no
signed result is ever silently accepted.

Anchor (primary enforcing site): `python/regolith/harness/attest.py :: verify_attestation`
