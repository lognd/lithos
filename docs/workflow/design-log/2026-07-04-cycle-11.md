## AD-17: the lowering pipeline (D82)

Escalated: nothing wired WO-07..13 together; `Session::check` only
parsed + rendered diagnostics, so BuildOutput carried no resolutions,
obligations, or snapshot hashes -- starving the golden corpus and the
WO-17 invariant suite.

DECISION (D82): a new PURE crate `rockhead-lower` between `oblig` and
`api` owns the end-to-end assembly driver. Layering becomes
`util <- diag <- qty <- syntax <- sem <- ir <- oblig <- lower <- api <- py`.
No IO, no rendering, never `Err`; six passes, one tracing span each:
`parse` -> `lower.entities` -> `lower.checks` -> `lower.contracts` ->
`lower.claims` -> `lower.discharge`. `check()` = passes 1-5;
`compile()` = 1-6 (static toy discharge). Per-subject INV-20 gating.
All IO stays in `Session`; the ONE renderer stays in `rockhead-api`.
Rejected: growing the driver inside `rockhead-api` (couples the FFI
surface to pass churn); Python-side assembly (violates AD-1). Recorded
normatively as AD-17; the work is WO-19.

BuildPayload becomes typed: `Vec<Resolution>`, `Vec<Obligation>`,
`Vec<SnapshotRecord>` (new), `evidence: Vec<Evidence>` (new). This
un-cuts WO-18 deliverable 6 (obligation-hash round-trip across FFI).

## AD-18: canonical encoder home (D83)

Escalated: AD-5 puts the canonical encoder in `rockhead-oblig`, but
`sem::EntityDb::snapshot_hash` needs canonical hashing and `sem` is
below `oblig`; it hashed serde_json, violating AD-6 "nothing hashes
JSON" (marked TODO(AD-6)).

DECISION (D83): move `canonical_cbor`, `EncodeError`, `content_address`,
and the workspace `SCHEMA_VERSION` constant down to
`rockhead_util::canon` (bottom of the stack); `rockhead-oblig`
re-exports all four unchanged, so no caller breaks. Schemas +
`export_schemas` stay in `rockhead-oblig` per AD-5. `sem`'s snapshot
hash migrates to `content_address("rockhead.sem.snapshot", ...)`.
Rejected: a dedicated hash crate (ceremony -- util is already the
determinism home); a documented sem/oblig split (two canonicalizers
desync silently, breaking INV-1/INV-10 key stability -- the exact bug
class the ledger exists to prevent). Recorded normatively as AD-18.
Migration must land BEFORE WO-19 records golden hashes (it changes
snapshot values; nothing durable pins the old ones yet).

## rockhead-lower module layout (WO-19 scaffold reference)

Crate deps: util, diag, qty, syntax, sem, ir, oblig + camino, serde,
tracing. Modules and public stub entry points (bodies
`todo!("STUB WO-19: ...")` per the stub convention):

- `lib.rs`: `lower(&[SourceFile]) -> LowerOutput`,
  `lower_and_discharge(&[SourceFile], &mut EvidenceCache) -> LowerOutput`,
  `parse_sources(&[SourceFile]) -> Vec<ParsedFile>` (non-stub).
- `output.rs`: `SourceFile{path,text}`, `ParsedFile{path,parse}`,
  `LowerOutput{diagnostics, resolutions, obligations, snapshots,
  evidence}`.
- `entities.rs`: `EntitySnapshots{scopes: IndexMap<String,EntityDb>,
  diagnostics, resolutions}`, `build_entities(&[ParsedFile])`.
- `checks.rs`: `CheckReport{diagnostics, orbits: OrbitTable}`,
  `run_checks(&EntitySnapshots)`.
- `contracts.rs`: `ContractGraph{interfaces, matings, budgets, systems,
  impls, diagnostics, resolutions}`,
  `build_contract_ir(&[ParsedFile], &EntitySnapshots)`.
- `claims.rs`: `ObligationSet{obligations, snapshots, diagnostics}`,
  `build_obligations(files, snapshots, checks, graph)`.
- `discharge.rs`: `discharge_static(&[Obligation], &ContractGraph,
  &mut EvidenceCache) -> Vec<Evidence>`.

`SnapshotRecord{scope, hash}` is a new `rockhead-oblig` boundary
schema; `Resolution`/`Cause` gain `JsonSchema`; `SCHEMA_VERSION`
1 -> 2.
