## F135. Artifact-presentation recon: the shipped sheets fail a professional bar in four ways (visual audit, evidence on disk)

The coordinator rendered representative shipped artifacts to PNG
and inspected them by eye (the D238 proof discipline, applied first
to the baseline). Every finding below is visible in the committed
demo outputs (`demos/out/`, regenerable via `make demos`):

1. MECH DRAWING (`demo3_removal_ribbed_panel/ribbed_panel_drawing
   .pdf`): dimensions are floating raw text (`bbox.depth=80.0000mm`
   painted over the view, no extension lines, no arrowheads, no
   dimension line); annotation text CLIPS off the right page edge
   (`height: 1...`); the title block is five unlabeled text lines
   (no field names, no date/sheet/units/projection cells).
2. OPT TRACE (`demo1_select_ebi_decode/opt_trace.pdf`): the trace
   is a naked polyline -- NO axes, ticks, labels, gridlines, or
   units; the winner annotation overflows the right page edge
   mid-hash; the candidate table is raw pipe-delimited text
   (`0|decoder_board...|2.4|True|ok`) overlapping the plot region
   and colliding with the title block.
3. CALC SHEET (`demo15_calc_audit/calc/j3_bearing__aff6c1f5010f
   .pdf`): the body is a raw `key|value` text dump that starts in
   the bottom third of an otherwise empty page, collides with the
   title block, and is CLIPPED mid-line at the page bottom (the
   value row is half-visible). The content (ISO 281 citation,
   provenance, margins) is excellent; the typesetting destroys it.
4. GERBER SET (`demo11_board_gerbers/boards/gerbers/`): copper,
   courtyard, edge-cuts, margin, job file -- and NOTHING else. No
   silkscreen on either side (no refdes, no polarity, no board
   name/rev), no soldermask, no paste, no drill file. No fab house
   would accept this set, and no technician could bring the board
   up from it.

Root-cause note: a drafting audit EXISTS
(`backends/drawings/audit.py`: title-block completeness, min text
height, no-overlapping-annotations, dimension completeness) and
passes all of the above -- so the audit rules are toothless against
exactly the defects they name. An audit that passes F135's evidence
is itself a bug (D238.1 makes it gating).

## D237. Debug instrumentation + hardware bring-up doctrine (AD-38; charter 40-debug-and-bring-up.md; WO-125..127)

Four rulings:

1. DEBUG IS AN EMISSION PROFILE, NOT A DIALECT. `regolith build
   --profile debug` / `ship --profile debug` augments EMITTED
   artifacts (board tap header + test points, firmware trace
   hooks, HDL tap taps) without touching design source, semantics,
   claims, or verdicts. The default (release) artifact set stays
   byte-identical when the profile is off -- determinism (AD-6/
   INV-10) holds per profile.
2. TAPS DERIVE FROM THE SAME TRUTH THE CENSUS READS. Every
   net/signal a claim names is a tap candidate; explicit taps are
   declared in the ship spec block's `"debug"` entry (the WO-102
   spec-block idiom -- no grammar change). Grammar-level `tap`
   statements are DEFERRED with a reopen criterion: a tap need
   that cannot be named by net path from the spec block.
3. THE BRING-UP HARNESS PACK IS AN ARTIFACT FAMILY (`dist/<proj>/
   harness/`): tap map, bring-up procedure document, capture
   configs for the analyzer, and an expected-signal manifest
   sourced ONLY from discharged claims / calc-book sheets /
   declared records (D224: never a fabricated expectation; an
   unverifiable expectation is a named absence with a reason).
4. THE TEST HARNESS HARDWARE IS ITSELF A LITHOS DESIGN. The
   logic-analyzer-class jig is authored in cuprite, ships through
   the same pipeline (census, calc book, gerbers, firmware), and
   mates the tap header via ONE shared pinout record (single home,
   AD-37 spirit) -- dogfooding is the acceptance test. New
   invariant INV-32 (tap-map/artifact agreement: every tap row
   exists in the emitted artifacts and every emitted tap is in the
   map) lands WITH its proof argument in WO-125.

## D238. Artifact presentation doctrine (AD-39; charter 41-artifact-presentation.md; WO-123/124)

Four rulings:

1. ONE NORMATIVE PRESENTATION STANDARD (charter 41) binds every
   renderer: sheet grammar (title block with named field cells,
   border, zone references, margins, typography scale), dimension
   entities (extension lines, arrowheads, units), tables are ruled
   tables (never pipe-delimited prose), charts carry axes/ticks/
   units/legend, and NOTHING clips or overlaps. The drafting audit
   becomes GATING and its rules are upgraded until they fail F135's
   evidence; new invariant INV-31 (shipped sheets are legible by
   construction) lands WITH its proof argument in WO-123.
2. THE FAB SET IS COMPLETE: silkscreen both sides (refdes,
   polarity/pin-1, board name + rev + design short-hash),
   soldermask, paste, drill, fab notes -- from BOTH the real-KiCad
   leg and the fake-KiCad tier (honest tier labeling unchanged),
   same file set either way.
3. PROOF IS VISUAL. A presentation change is done only after the
   coordinator renders the artifacts and records acceptance
   (artifact hashes + verdict) in the design log. Automated audits
   gate regressions; a human eye grants "gorgeous".
4. INFORMATIONAL DENSITY WITH PROVENANCE: every sheet carries the
   design content address, schema version, style pack, and rev;
   every number printed traces to the payload/calc book (D224
   extension) -- density never trades against the legibility rules
   in ruling 1. Determinism holds: no wall-clock timestamps on any
   artifact.
