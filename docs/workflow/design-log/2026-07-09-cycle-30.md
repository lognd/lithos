## D159. One optimization engine, riding the pipeline (AD-30)

The engine lands as `regolith.orchestrator.optimize` (Python: search
is scheduling/iteration, AD-1), ONE home with two drivers:

- **Discrete**: conflict-driven greedy search per regolith/07 sec. 7
  -- policy-ordered candidate descent screened by the cheap tier,
  lazy full discharge, blame-set backjumping, nogoods persisted in
  the existing `NogoodCache`.
- **Continuous**: bounded-domain numeric refinement for
  `in [lo, hi]` variables, generalizing (not duplicating)
  `lazy_loop` -- strategies are IN-HOUSE, deterministic, seeded
  (golden-section for 1-D, Nelder-Mead simplex for N-D); no
  third-party optimization framework enters v1.

Load-bearing rules (each argued against the mantras):

1. **Candidates are evaluated ONLY through the real pipeline**
   (`build`/`staged_build` + `discharge_all`) -- never a private
   scoring path (AD-22; Unambiguous: the score IS the evidence).
2. **Objectives come ONLY from the existing surface** (per-variable
   direction, `policy: minimize` lexicographic order, budgets as
   feasibility) -- no new objective grammar (one word one idea).
3. **The search trail is evidence**: an `OptimizationTrace` payload
   (Rust schema, AD-5/AD-18) records candidates, objective vectors,
   verdict summaries, seed, strategy id+version, budget spent, and
   the winner -- content-addressed, cached, resumable.
4. **Winners are pinned with a cause**: lockfile rows
   `cause: optimize(<objective>, trace=<digest>)` (INV-21); a
   number chosen by search names why it changed, like every other
   resolved value.
5. **Determinism**: same sources + records + seed + budget => same
   winner and byte-identical trace (INV-10 extended). The guarantee
   enters regolith/13 as INV-30 WITH its proof argument in the same
   change as the machinery (house rule).
6. **The safety property is untouched**: optimization proposes
   inputs and re-keys obligations; it can never relabel a verdict
   (INV-2 ladder argument, verbatim from loop.py's contract).

Rejected: a standalone optimizer tool (second margin rule, second
evidence model -- the AD-29 argument verbatim); scipy.optimize as
the driver (nondeterministic-across-platform line search internals
vs. INV-10; our domains are small and our evaluations dominate);
exposing search internals as API v1 (CLI + trace artifact are the
consumer surface).

## D161. `by select(...)`: the sixth impl strategy (declared alternatives)

The EBI-style implementation choice gets a source surface: an
artifact may be implemented `by select(<impl-ref>, <impl-ref>, ...)`
-- a closed, human-declared candidate list, the sixth impl strategy
beside `by spec / composing / circuit / vendor / extern`
(regolith/08 sec. 4). Semantics:

- Every candidate must independently satisfy the contract's static
  tier (the integer-domain monomorphization rule applied to impls:
  "valid for the whole domain or shrink the domain").
- The choice lowers to a `ChoicePoint` payload; the D159 discrete
  driver decides per policy objectives; the winner pins with
  `cause: optimize(...)`; non-winners stay in the trace.
- Sovereignty is preserved (AD-28 untouched): the human authored the
  candidate set; `advise:` recommendations still never
  auto-substitute. `select` with one candidate is legal and inert
  (degenerate pin).
- `policy: prefer/forbid` apply to candidates exactly as to any
  allocation domain.

Spelled `select` (not `any`/`oneof`): the tool SELECTS, the author
declares -- intent-based, and unambiguous because the list is closed
and each member independently checked. Mantra check: Unambiguous
(closed list, per-candidate static checks), Intent-based (defer the
decision, not the design), User-friendly (one keyword, no new block
form). Grammar/CST/lowering land in WO-56; regolith/08 sec. 4 and
track vocabularies update in the same change.

## D162. Expensive realized-domain optimization is the continuous driver over the staged loop

Geometry (CFD-propeller class) optimization = the D159 continuous
driver where each candidate evaluation runs the FULL staged loop
(lower -> realize -> re-lower, AD-25) and discharges through
whatever models the harness selects -- including external AD-19
solver packs. Nothing geometry-specific enters the engine. The
expensive-evaluation discipline:

- An explicit **evaluation budget** (max evaluations and/or wall
  time) is mandatory at invocation (CLI flag / config profile);
  exhausting it yields the best-feasible-so-far PLUS an honest
  indeterminate trace status -- never a silent stop.
- The trace is a **checkpoint**: `regolith optimize --resume`
  continues from it; every candidate's evidence is content-address
  cached, so resumed or re-run searches re-evaluate nothing
  unchanged (incrementality falls out of AD-18, proven by test).
- CFD-class evaluators are ordinary model packs (AD-19), NEVER
  in-tree physics (feldspar's non-goal discipline applies); v1
  acceptance uses existing closed-form models so the corpus stays
  runnable in CI.

## D164. One configuration doctrine: global file + project manifest, source-attributed

Configuration the TUI/CLI edits has ONE precedence chain, weakest
first: global user file (`~/.config/regolith/config.toml` via
