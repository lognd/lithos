## A. Invariant audit -- holes found and fixed

The full ledger with proof arguments is `substrate/13-invariants.md`
(D38). Holes found by trying to prove each claimed invariant:

**F46. `@hint` droppability was undefined -- and unsound as read.**
"Droppable verification hints (symmetry, regime)" allowed a reading
where a model uses a user hint to justify domain validity or coverage
-- a wrong hint could then discharge a false claim. Fix (substrate 07):
**hints are never load-bearing** -- a hint may affect only what is
tried first (candidate order, starting points), never verdicts;
formally, for a fixed resolved design, verdicts are hint-invariant.
Anything a model needs for *validity* must come from checked sources
or be an explicit `assume!`. Consequently, entity-DB symmetry is
reclassified: it flows into obligations as **checked facts**
(load-bearing allowed -- it is computed conservatively and can never
assert a false symmetry), not as hints.

**F47. Waiving a rule that resolves a `free` variable was incoherent.**
A DFM rule is both a check and a resolver (`radius=free` resolves to
its minimum). Waiving `dfm(min_bend_radius)` on an entity whose radius
is `free` leaves the variable sourceless. Fix (substrate 12): a waiver
targets a rule's *verdict*, never its resolution duty; if the waived
scope contains a variable the rule resolves, that is an error whose
fix is asserting the value (rung 1) alongside the waiver.

**F48. Boundary subsumption needed its phrasing rule.** Containment is
the safe direction for every boundary kind only because boundary
entries are, by definition, *tolerated envelopes* (what the artifact
was proven under). One sentence added to substrate 04 sec. 5.6; no
behavior change.

**F49. Verify-one-instantiate-n was unsound for asymmetric loads.**
A geometrically perfect bolt-circle orbit under a bending moment does
NOT license verifying one bolt: the claim's *givens* are not
orbit-invariant. Fix (substrate 07): orbit extension is legal iff the
obligation's givens are invariant under the orbit's group -- checked
by the discharging model (which may instead use the orbit-worst-case
envelope, as the joint-diagram effects already do); otherwise
per-instance discharge. Entity-DB symmetry (F46) covers the subject;
this covers the loads.

**F50. Bit-reproducibility over-claimed.** FP nondeterminism in
kernels/solvers is real. Fix (substrate 09): reproducibility is
guaranteed for *decisions and evidence identities*; numerical evidence
values are reproducible per the model's declared `deterministic:`
flag, and nondeterministic models must fold seeds/settings into the
evidence hash inputs -- so cache identity never lies.

**F51. Computed semver vs non-literal promise slots.** Slots valued
`derived`/`allocated` in a published contract cannot be
interval-diffed; they compare structurally (presence/type), and only
literal-valued slots participate in widen/narrow comparison. One line
in substrate 11 sec. 5.

**F52. "Base evidence remains valid under every target" was false as
stated.** Adding a debug LED perturbs placement/routing, which
perturbs extracted parasitics, which invalidates base SI evidence.
Fix (substrate 04 sec. 6): contract-level base evidence is always
valid (targets cannot touch contracts); realization-level base
evidence is reused iff the base realization snapshot is unchanged,
which the new rule guarantees by construction (target-added content
realizes only inside reserved regions, base realization reused
verbatim); where a target genuinely perturbs the base, content
addressing forces re-verification -- reuse is never silent.

**F53. Unscoped waivers could silently absorb NEW failures.** A waiver
declared today matches a failure introduced next month. Fix
(substrate 12): the lockfile records each waiver's matched set; growth
relative to the previous lockfile surfaces in the diff and as a build
warning naming the new members.

**F54. Deviations dodged trust floors.** An evidence-carrying waiver
(deviation) is evidence like any other: it now must meet the `trust:`
floor of the claim group it waives, or the waiver stays release-gated.

**F55. D25's latency implication was under-conservative.** "Workload
latency <= intent latency" ignores transport: an intent's latency is
end-to-end over its flow chain. Fix (elec 05): reconciliation runs
through the flow ledger's latency budget -- workload latency + realized
transport latencies <= intent latency, same machinery as accuracy
chains. (An audit catch against a decision I made in cycle 2.)

**F56. `allocate:` vs `policy:` adjacency.** Budget `allocate:`
remains the *local share-splitting policy inside one budget*; `policy:`
objectives are global and lexicographic. One clarifying sentence in
substrate 12 sec. 4.

## C. Decisions

**D38. The invariant ledger** -- new `substrate/13-invariants.md`:
twenty-six invariants (INV-1..26; user directive mid-cycle: state ALL
of them, including the founding guarantees that had only lived as
prose -- type soundness, reference determinism, promises-not-actuals,
check gating, resolution provenance, foreign-content pinning, region
exclusivity, release-gate totality, coverage honesty, and the defaults
test as a meta-invariant). Each has statement, enforcing mechanism,
proof argument, and test family. WO-17 turns the ledger into an
executable suite; the ledger is normative -- a spec change that breaks
a proof argument must update the ledger in the same change. Two
invariants are deliberately flagged as *per-model obligations* rather
than global proofs (INV-9 corner maps, the model-shape half of
INV-25): that honesty is itself the point.

**D39. EOPEN-7 closed.** Formal sketch added (elec 03 sec. 1a):
cross-domain simultaneous-event ordering is unobservable because
domains share no state except through crossing matings (which impose
handshake/ordering semantics) and converter ports (non-instantaneous
by construction); a trace-equivalence argument over the composed
system's observable ports. Documentation debt paid.

**D40. Waiver expiry.** Optional `expires: <date>` on `waive`; an
expired waiver behaves as absent (the failure returns) and the waiver
itself becomes a stale-waiver error until removed or renewed --
concessions cannot quietly outlive their justification. Chosen over
revision-bound expiry ("until rev C") because revisions are not a
language concept; dates are unambiguous and diff-visible.

**D41. EOPEN-17 hardened (still [LEANING]).** IO banking added to the
host-binding constraint shape: each port's IO standard/domain must be
satisfiable by some bank assignment (a sub-problem of the same
capability matching); the frame-grabber example now shows a
`hosted_on` pin plus a bank-driven constraint. Second full example
still carried.
