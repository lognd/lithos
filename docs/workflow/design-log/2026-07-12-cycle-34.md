## F121. Emission-pipeline survey: the ten corners (2026-07-12)

A full survey of the emission surface (preview/ship, staged_build,
producers, renderers, gate) found these load-bearing weak points:

1. The "real" KiCad wrapper draws a fixed 50mm placeholder square
   (`realizer/elec/kicad_wrapper.py`); gerber export unproven
   end-to-end against a real board.
2. Mech drawings render a bbox-rectangle stand-in, never projected
   geometry, despite the IR supporting arcs/polylines.
3. BOM is 100% caller-supplied; `mass_hint` is an AREA
   (`geometry.topology.area_mm2`) labeled as a mass hint.
4. Costing is evidence-only: no cost sheet or schedule artifact
   exists, though AD-27 names schedules in scope.
5. Firmware and HDL are realized/verified but never ship -- no
   backend exists for either.
6. Assembly instructions cannot topologically sort: RealizedAssembly
   carries no mate edges; `mate_ref` is always None.
7. The renderer quintet is hard-coded in `files_for_model`; no
   renderer extension seam exists (backends have one, formats do
   not).
8. Drawing `source_digest` is a local blake3 over IR JSON, not the
   canonical content address.
9. Native artifact bytes (STEP/kicad_pcb) are only transiently
   available; realizers do not persist them, so `ship --build` can
   fail on geometry it itself realized.
10. `model_for_spec` is an if/elif ladder and `auto_specs`
    hard-codes track wiring: a new artifact type is an invasive
    multi-site change.

## D206. Ship-green means INV-24 as written; the campaign law

Ruling (the load-bearing one): "every example ships" is achieved by
completing INV-24's designed semantics -- a green release means
"proven, or explicitly accepted by a named human reason, nothing
between" -- NEVER by weakening verdicts. Concretely:

1. Discharge everything the machinery can honestly discharge
   (models, records, geometry, chaining).
2. Accept the genuinely-unbounded residuals via the rung-6/7
   ladder (regolith/12): source-level `waive` with mandatory
   `basis:`, evidence-carrying where evidence exists.
3. The D195/F116 law stands untouched: no fabricated conformance
   windows, no vacuous discharges. A `conformance_windows_
   unresolved` obligation on a corpus design is exactly the shape
   `waive` exists for: the design's author accepts the open edge
   and says why.
4. Verdict math, trust floors, INV-2/12/13/24/26/28 are all
   untouchable. The gate gains CONSUMPTION of the existing ledger,
   not new powers.

This supersedes nothing: D195 deferred the QUESTION of inventing
windows (still refused); it never forbade the acceptance ladder,
which predates it (cycle 3).

## D207. Evidence class for corpus deviations: engineering memos

A bare waiver stays release-gated (per-item CLI acknowledgment is
exploration-only, regolith/12 rule 9), so a non-interactive fleet
ship needs deviations. Ruling: an in-project engineering memo --
a markdown document, hash-pinned exactly like any catalog doc --
is a legitimate `by doc(<ref>)` evidence form (the spec's "vendor
letter" class). Memos live in the project (`memos/*.md`),
resolve through the D192/D201 record-path machinery, confer
community tier (INV-14: unsigned content = community), and trust
floors still bind -- a claim demanding `trust: >= certified`
cannot be memo-waived. The memo must state: the obligation set
accepted, why the residual is genuinely unbounded (citing the WO
wall or design-log finding), and the discharge path that would
retire the waiver. Expiry is encouraged, not mandated, for corpus
examples.

## D209 (answers D205's open design). Optimizer evaluators come from the discharge pipeline

The WO-97 blocker was "no production mechanism builds an evaluator
from a part's declared claim." Ruling: there is no new evaluator
concept -- the evaluator for a bounded geometric slot IS the
discharge pipeline specialized per candidate, exactly the WO-57
staged-evaluator precedent generalized from discrete choice_points
to bounded scalar slots: candidate value -> givens overlay on the
obligation's given set -> the claim's model invocation -> margin.
Feasibility = margin >= 0 on every claim naming the slot's part;
objective = the slot's declared direction (minimize/maximize the
slot value itself). If the claim's model defers, the optimize
defers with `optimizer_evaluator_deferred` naming the model gap --
honest, never a fabricated closure. `SegmentLength::Bounded` rides
WO-104's single schema bump (28->29). WO-97 is dispatchable.
