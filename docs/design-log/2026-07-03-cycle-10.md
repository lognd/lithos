# Design Log -- Cycle 10 (2026-07-03)

Implementation kickoff plus an owner-directed name change. No spec
changes (hematite 0.13 / cuprite 0.10). Findings continue F-numbering;
decisions continue D-numbering (D81).

## Context

Cycle 9 settled the architecture (00-architecture.md, AD-1..16) and

## WO-01 (scaffold) -- built and green

Hybrid Rust/Python workspace stood up exactly per AD-2/AD-4/AD-8:

- Cargo workspace, nine crates (`rockhead-util` .. `rockhead-py`) in
  the strict layering `util <- diag <- qty <- syntax <- sem <- ir <-
  oblig <- api <- py`, each with a module docstring naming its
  substrate doc and a placeholder unit test.
- maturin + uv single-wheel layout; the `rockhead._core` abi3-py312
  extension compiles and imports from Python.
- Typed facade (`rockhead/compiler.py`) as the ONE door to `_core`;
  `make check` greps for any other importer (AD-4 enforced).
- pyo3-log bridge proven end to end: a Rust `tracing` event (with the
  `log` feature) reaches Python `logging` (test runs in a subprocess
  with `ROCKHEAD_LOG=DEBUG` so pyo3-log caches the level; logs render
  on stderr, stdout stays clean for command output).
- Makefile (AD-13 targets), CI skeleton (AD-12), cargo-deny, gitignore.
- `make install && make check` green on linux.

Two small deviations from the WO text, both improvements: logs go to
stderr (not stdout) because for a compiler CLI stdout is the data
channel; the per-call version log sits at DEBUG (a hot accessor at INFO
is noise). Recorded here so a future reader does not "fix" them back.

## D81 -- the name change (owner's decision)

The owner replaced the machine-metaphor names with a single geology
theme, and filled the open umbrella slot:

| role | was (D78) | now (D81) | extension |
|---|---|---|---|
| mechanical language | mill | **hematite** | `.mill` -> `.hem` |
| electrical/computer language | loom | **cuprite** | `.loom` -> `.cupr` |
| package tool | quarry | quarry | -- |
| registry | lodestone | lodestone | -- |
| umbrella dist/import/CLI | (open; `decl`) | **rockhead** | -- |

Rationale (recorded because it supersedes the cycle-8/9 machine
argument, which stays in those logs as history):

- **Coherent theme.** quarry, lodestone, rockhead, hematite, cuprite
  are all mining/geology. hematite is iron ore -> steel/structure ->
  the mechanical language; cuprite is copper ore -> wire/current ->
  the electrical language. The metaphor now maps onto the domains
  (iron for structure, copper for current) instead of onto shaping
  machines.
- **Extension collisions.** `.loom` collides with the established
  loompy/AnnData format; `.mill` was metaphor-only. New extensions
  were chosen short and collision-free: `.hem`, `.cupr` (rejected
  `.cup`/`.cpr` -- SeeYou/Cubase).

## Sweep discipline (how the rename was executed safely)

The words collide with domain and English vocabulary, so a blind
find/replace was rejected:

- `milled`, `milling`, `cnc_mill`, and the machining noun `mill`
  (`lathe+mill`, `Face(mill+lap)`) are DSL/domain vocabulary and were
  preserved; only the language-name `mill` became `hematite`.
- `declare`/`declaring`/`declared` are English; `\bdecl\b` (the
  placeholder toolchain name) never matched them.
- **Excluded from rewriting:** `docs/archive/` (charter-verbatim) and
  `docs/design-log/` (dated historical ledgers -- the cycle-8/9 entries
  argue the mill/loom machine metaphor, which was true when written).
  History is appended (this file), not edited.

Scope swept: the code tree (crates, python package, manifests, tests,
CI), the living docs (`README`, `substrate/`, `mech/`, `elec/`,
`implementation/`, `TODO.md`), and `examples/` including file-extension
renames (`*.mill` -> `*.hem`, `*.loom` -> `*.cupr`). The extension
registry (`rockhead-syntax`) recognizes only `.hem`/`.cupr`.
