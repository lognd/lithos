## A. Decisions

**D30. The override ladder + in-source `waive`.** [SETTLED]
New substrate doc `12-overrides-and-hints.md`. One doctrine unifies
every mechanism by which a human redirects the system, ordered by how
much the human claims to know:

1. **Assert** -- write a literal (value-source grammar). Claims truth;
   every check still runs against it.
2. **Pin** -- the lock family (`locked:`, `use`, `sequence:`,
   `merge()`, `hosted_on`): fix a decision the system would otherwise
   make. Checks unchanged; a pin can cause a spurious failure, never a
   silent pass.
3. **Hint** -- `@hint`, `prefer`: droppable guidance; changes what is
   tried first, never what is true.
4. **Override data** -- `override <record> by <evidence>`: replace a
   registry fact with better local knowledge; evidence mandatory;
   checks re-run on the new data.
5. **Force a model** -- `model=<impl>` generalized to any claim: pin
   the discharge path. Margin math is unchanged, so a forced cheap
   model that cannot close the margin yields `indeterminate`, never a
   pass -- forcing cannot forge evidence.
6. **Assume** -- `assume!(expr, basis=)`: accept an obligation without
   evidence. Ledgered; release-gated.
7. **Waive** -- NEW in-source construct: accept a specific violated or
   indeterminate result, scoped, with mandatory basis:

   ```
   waive Manufacture.makeable on milled.deep_bore:
       basis: "wire-EDM vendor confirmed; quote Q-2214"
       by test(first_article_fai_112)      # optional evidence clause
   ```

   Targets are named claims (`Group.claim`) or rule-pack rules
   (`dfm(rule_id)`, `drc(rule_id)`) scoped to entities with `on`.
   A waiver **with** an evidence clause is a *deviation* -- permitted
   in `--release`, still ledgered. A waiver **without** evidence is
   release-gated like `assume!` (per-item CLI acknowledgment). The
   CLI `--waive` remains for exploration but never persists: promoting
   it to source is the only durable form, so every waiver is
   diff-reviewable at a source location with an author and a reason.

**The safety property** (the load-bearing sentence of the doc): *no
mechanism on the ladder can convert `violated` into `discharged` --
overrides supply better inputs, narrow choices, or record accepted
risk; evidence math is untouchable.* The audit surface is fourfold:
source construct (diff-reviewable), lockfile entry, evidence ledger,
`--release` gates.
- *Unambiguous:* one ladder, each rung one construct; the same
  question ("who decided, on what basis?") has one answer form
  everywhere.
- *Intent-based:* rungs 1-3 express knowledge and preference; rungs
  6-7 express accepted risk -- both are intent, both are declared, and
  the system still derives all consequences it can.
- *User-friendly:* the expert is never fought, only recorded; the
  reviewer reads waivers in the diff, not in a wiki.

**D31. SOPEN-4 resolved: `policy:` blocks (prefer / forbid / global
objectives).**
```
policy:                         # system/assembly level (or per budget)
    prefer vendor(ti) over vendor(onsemi)
    prefer package(sot23)
    forbid package(bga)
    minimize distinct_values    # global, soft, lexicographic after
    minimize total_cost         #   all claims are satisfiable
```
- `prefer` is **soft**: reorders candidate exploration in the
  conflict-driven search; can never cause a failure; when a preference
  is decisive for a pinned outcome the lockfile row carries
  `policy: prefer(...)` alongside the cause (defaults-test third
  prong).
- `forbid` is **hard**: intersects allocation domains like a structure
  boundary; resulting infeasibility names the forbid.
- Global objectives (the retired `minimize(count_distinct)` need) live
  here, at policy altitude, not on variables. Applied lexicographically
  in declared order, strictly after claim satisfiability.

**D32. EOPEN-7 resolved in shape: event-bounded hybrid semantics.**
Forced by `examples/elec/sampled_buck.loom` (continuous plant +
sampled control loop -- the case cycles 1-2 lacked):
1. **Within a clock domain: synchronous-reactive.** All `on <event>:`
   bodies of a domain fire atomically at the event instant; `<=`
   commits at instant end (coherent pre-state); combinational `=`
   networks must be acyclic (compile error otherwise).
2. **Continuous subset: DAE evolution between event instants.**
3. **Coupling exclusively through converter ports** (`adc`,
   `comparator`, `dac`, `pwm`, clocked `digital` drive), each
   declaring its sampling/update event and its error contribution.
   Reading `v(node)` inside a clocked body without a converter port is
   a compile error naming the port kinds -- the CDC rule's
   continuous/discrete analog.
4. **Converters are non-instantaneous by construction** (sample takes
   the pre-instant continuous value; updates take effect after the
   instant, ZOH-style). This breaks algebraic loops across the
   boundary structurally -- no Zeno/causality analysis needed for
   legality.
5. Continuous-to-event: `v(x).crosses(th)` derived events (already in
   the quantity core) are the zero-crossing mechanism.
Prior art: Ptolemy's CT+SR composition, hybrid automata; the delta
here is that the *type system* (port kinds, domains) enforces the
composition discipline rather than convention. Remaining under
EOPEN-7: the formal write-up (simultaneous-event tie-breaking across
independent clock domains -- resolved operationally by "domains only
interact through crossings/converters, so ordering is unobservable";
needs a proof sketch, not a design change).

**D33. OPEN-13 closed: weld models + DFM rules.**
- Harness pack `std.mech.weld` (two-halved): signature

  resultants; outputs: throat stress components; domain: rigid-plate,
  elastic) implemented by the line-weld (welds-as-lines) method as
  the cheap tier; shell-FEA node as the expensive tier. Margin-driven

- Weld DFM rules (eager, per weld process module): torch access

**D34. OPEN-1 closed: variants ride the swept-obligation machinery.**
A `variant` axis is a discrete config domain; the compiler emits ONE
obligation carrying the domain (`sweep: hand in {left, right}`) --
SEAM-3 verbatim. Coverage rules follow the model: a model that can
declare mirror/permutation equivalence between variant points
discharges one and extends (the symmetry machinery); otherwise it
sweeps, with per-point caching, so shared geometry across variants is
shared evidence by content address. Per-variant lockfile sections as
already decided. Nothing new was needed -- the resolution is the
observation that two existing mechanisms compose.

**D35. EOPEN-17 shape [LEANING].** Host binding = capability matching,
same machinery as pin-mux: synthesis-estimate demand vectors (LUT/FF/
BRAM/DSP, IO count per standard) against the part record's resource
table; IO banking as per-bank domain/standard capability. `hosted_on`
pins it; lockfile `cause: planner(host ...)`. Stays open pending a
second worked FPGA example.

**D36. Pins-in-source have exactly one home: `locked:`.**
The "match an existing connector" case gets its spelling:
`locked: pinmux(u_mcu.uart2.tx): pa2` on the board/system. This is
the only position where a package pin name may appear in design
source -- deliberate, auditable, lockfile-mirrored (elec `04`).

**D37. Manual lowering and external linkage.** [SETTLED in shape;
user directive mid-cycle]
The decision ladder (D30) gets its representational twin: at every
lowering edge, the human may hand-write the lower level or link
foreign content -- and the system's job collapses from *deriving* the
lower level to *checking conformance* of the supplied content against
the upper level's contract. Synthesis is expensive and heuristic;
checking is cheap and sound -- the compiler/linker model (headers
declare, the linker checks the boundary). New section in substrate
`08-lowering-architecture.md`:

- **`by extern(ref, format)`** is the fifth impl strategy (beside
  spec/composing/circuit/vendor): a foreign artifact implements this
  contract and *stays foreign* -- own hash, own lifecycle, possibly
  opaque (encrypted IP). Distinct from `import(path)` stages, where
  foreign data becomes THIS artifact's realized state in the pipeline.
  Link-vs-merge, like `extern` symbols vs `#include`.
- Entry points per level: L2 = `vendor(ref)` (the existing form IS
  L2 linkage); L3 = transparent formats elaborated into design IR
  (Verilog/SPICE netlists for blocks, DXF outlines for profiles) with
  full static checks + equivalence obligations; L4 = `import` stages
  and opaque externs (compiled netlists, prebuilt firmware ELF,
  encrypted IP) with T2-where-measurable + evidence for the rest;
  L5 = `by test(ref)` (supplied evidence was always extern-at-L5,
  now named); L6 = supplied plans (`plan: extern("op10.nc",
  gcode_fanuc)`) verified by the planner in **check mode** (verify a
  given plan: collision/reach on supplied toolpaths) -- cheaper than
  planning, same evidence type.
- **No dead uppers** (the anti-shadowing rule): when both levels are
  written, the lower must conform to the upper -- a hand-written
  realization that contradicts its spec is a failing equivalence
  obligation, never a silent spec-shadow. If only the lower level is
  written, contracts are authored there (the brownfield posture).
- Format readers ship as packages (new `formats` kind in substrate
  `11`), hash-pinned -- linkage is reproducible.
- *Unambiguous:* one linkage keyword, one merge keyword, a stated
  link-vs-merge distinction; no dead specs. *Intent-based:* the
  contract stays the source of truth; foreign content is checked
  against it, not around it. *User-friendly:* generate-then-check is
  also exactly the LLM workflow the charter promises.

## B. Findings

**F42. `--waive` was an audit hole.** The only waiver mechanism was a
CLI flag: not in the diff, not attributable, not scoped. D30's
in-source `waive` fixes it; the CLI form is demoted to exploration
(non-persisting).

**F43. Escalation opt-ins are ladder rung 5.** `model=fea_contact`,
`stiffness=measured` (mech 03) and `model=spice_extracted` (substrate
04) were already the "force a model" idea; substrate/12 names them as
such, and generalizes the pin to any claim. No behavior change.

**F44. The sampled-buck example needed a discrete-time claim form:**
`elec.phase_margin(loop(<probe> -> <actuator>)) >= 45deg` -- loop
stability as an ordinary claim, discharged by discrete-time
loop-analysis models (cheap: averaged small-signal; expensive:
time-domain limit-cycle simulation). Added to the elec claim
vocabulary as a sketch; harness models are future work.

**F45. The gearbox exercised `mesh_alignment` in anger** (the one doc
budget no example had used) and needed nothing new -- budget, members,
`locked:` entries, and the E0432 story compose as documented. It also
demonstrates rungs 2/3/4/7 of the ladder in one file.
