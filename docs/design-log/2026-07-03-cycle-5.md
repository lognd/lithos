## A. Decisions

**D42. Claim-position two-sided comparator: infix `within`, no `=`.**
[SETTLED; decided by grammar experiment]
Two examples had invented `mech.backlash(mesh) = within [0.05mm,
0.15mm]`. The spelling was never defined for claim position. A
tokenizer experiment over the real claim corpus (scratchpad,
`within_experiment.py`) showed the `= within` form gives `=` five
context-dependent meanings on claim lines (kwarg, config-bind,
discrete equality, spec relation, window-intro) and forces the
parser to classify the claim's semantic class from the right-hand
side; the infix form adds `within` to `< <= > >=` as a one-token
comparator, adds no meaning to `=`, and one character of lookahead
separates all `within` positions (`(` tol-ball, `[` window, digit
time-window). Canonical: claims write `expr within [lo, hi]`; slots
keep comparator literals after `:`/`=` as value sources (`static: <=
12N`, `k: within [0.8, 1.6] N/mm`) -- same slot-vs-expression split
as every other comparator. (Substrate `03`; mech `04` sections 2G/5;
molded clip + gear reducer fixed.)
- *Unambiguous:* `=` in claims stays one idea (discrete equality).

**D43. Artifact-position imports get an implicit stage named `src`.**
[SETTLED; the examples already assumed it]
`parts: plate: import("baseplate_rev2.step") sealed` was used by
pillow_block and torch_igniter, whose retro-contract impls bind
`src.holes.where(...)` -- but no doc defined where `src` comes from.
Rule: an import bound directly in a parts list is shorthand for an
artifact with exactly one import stage, named `src` by rule; inside
`impl ... for <that-part>:` bindings qualify with `src.`, from outside
the part name qualifies as usual. (Substrate `06`; mech `02` sec. 2.)
- *Unambiguous:* one defined name, not a convention the examples
  happened to share. Alternative (require a full `part` wrapper per
  import) fails User-friendly for the brownfield wedge.

**D44. Image `partitions:` use `remainder`; open-ended `[a ..]`
ranges retired.** [SETTLED]
flight_controller wrote `log: flash[1MB ..]` with a comment saying
"remainder"; fpga wrote `app: flash[32kB ..]`. That is a second,
undocumented spelling of the complement, and it requires knowing the
extent's end. The binding table already says partitions follow the
zones `remainder` rule -- so they use it literally: `log: remainder`
(the complement, exactly once, computed size). `[i .. j]` stays fully
closed-form.
- *Unambiguous:* one spelling of "everything else" across zones and
  partitions (they are one substrate concept, F26).

**D45. Projects, files, and teams.** [SETTLED in shape; user
directive]
New substrate `11` section 9. Almost all of it is existing decisions
composed: project = manifest + source tree + one lockfile (no ambient
registry state); contract-first decomposition is the parallel-work
unit (L2 verifies with zero artifacts; each artifact implements its
contract in its own files); obligation ownership follows declaration
(D24), and content-addressed evidence makes a team-shared or CI
evidence cache sound by construction (keys decide reuse, not trust);
merges reconcile in source and the lockfile regenerates
deterministically (INV-10), so lockfile conflicts resolve by rebuild
and the post-merge lockfile diff is the review artifact; stabilized
subtrees graduate to `parts`-kind packages by changing one import
line. Three genuinely new rules:
1. **Files are containers, not scopes** -- file boundaries carry no
   semantics beyond import visibility. Stated as **INV-27
   (file-layout invariance)** with mechanism and proof argument in
   the ledger, same change (house rule): resolution binds to
   declaration identity, obligation keys contain no source paths, so
   verdicts/lockfile rows/evidence keys are invariant under how
   declarations are split across files. WO-17 gains the file-split
   identity-diff family.
2. **Import forms**: named imports bind declarations; bare package
   imports load registry contributions only (resolved by coherence);
   path imports must name what they take (a bare path import would
   splice a teammate's whole namespace in, unreviewed). This also
   legalizes what the examples already did (`import std.intents`
   bare; every path import named).
3. **The import graph is acyclic**; the constructive fix for a real
   cycle is importing the shared interface pack instead of each
   other.
- *Unambiguous:* file layout provably cannot change meaning;
  *Intent-based:* teams organize by ownership, not language rules;
  *User-friendly:* blast radius of a teammate's change is computed
  (INV-1/INV-19), not feared.

**D46. EOPEN-4 retired: no waveform constraint systems.** [SETTLED]
"Prototype or retire" resolved by attempting the forcing example. A
parameterized power-sequencing mask is expressible today:

```
mask seq_rise<t_ramp: time = 5ms>:
    from_fn(t): [0.9 * vdd_final * min(t/t_ramp, 1), 
                 1.1 * vdd_final * min(t/t_ramp, 1) + 50mV]
```

Eye masks parameterized by unit interval, ramps by slew -- every
observed case is `from_fn`/`from_table` plus parameters. The mech
walk + DOF ledger analog (segments *solved for* jointly under claims)
has no failing example demanding it, and masks are consumed as
envelopes, not solved artifacts. Retired; reopen only on a mask whose
segment geometry must be an optimizer output. (Elec `03` sec. 7;
EOPEN list updated.)
