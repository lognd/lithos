## D130. Wetted-path data is DECLARED design intent, not derived geometry

**The problem.** The extract seam (`regolith-lower::extract`,
WO-32 D2) reads, per routed segment: flow area, centreline length,
bend (angle, radius), a process-capability roughness class, signed
elevation change, and optionally a thin-wall record (E, thickness,
diameter). Which voids of a solid are WETTED, what the wall
material's modulus is, and how a cavity decomposes into 1D routed
segments are design intent -- a B-rep solid does not carry them,
and reconstructing a 1D hydraulic network from raw geometry
(centerline extraction) is research-grade guessing. The realizer
must never guess (WO-22's "total and honest" posture).

**The decision.** The wetted-path decomposition enters the
realizer the same way every other resolved fact does: declared in
its input contract. `FeatureProgram` (the realizer's ONLY input,
AD-4) is extended -- `FEATURE_PROGRAM_SCHEMA_VERSION` 1 -> 2:

- NEW part-level `flow_paths: tuple[FlowPath, ...]`. A `FlowPath`
  is `selector` + ordered `segments`. Selector convention for
  mech-emitted paths is pinned: `<stage_name>.wetted` (matches the
  WO-32 hand fixtures' `coolant.wetted`).
- A `FlowSegment` declares: `role` (the seam's per-segment
  environment slot); an optional `bore` reference naming the
  feature op (stage + feature name) that cuts it, for realizer
  cross-validation; resolved `flow_area`, `length`,
  `elevation_change` (Cause-tagged, like every resolved param);
  optional `bend` (angle, radius); `roughness_class` as a label
  from THE process-capability table (`regolith-lower::extract`'s
  `ROUGHNESS_TABLE` -- the single home stays where it is; the
  producer declares a label, the seam validates it, an unknown
  label stays the existing typed `ExtractError`); and an optional
  `wall` (thickness, diameter -- geometry only; E comes from the
  material record below).
- NEW `material_props: MaterialProps | None` -- resolved material
  property VALUES (Young's modulus at minimum; density optional),
  Cause-tagged, resolved PRODUCER-side. WO-22 cut #2 stands: the
  realizer owns no physics table; it combines declared E with
  declared wall geometry when emitting a segment's wall record.

**The realizer's v1 duty** is validate-and-emit, never
derive-and-guess: check every declared segment against the
realized solid where the geometry fixes the answer (the referenced
bore exists; a bored segment's flow area is consistent with the
bore's resolved diameter; cavity-volume cross-checks once the
`.cavity` surface lands), then emit the declared measures into the
realized IR as intervals (degenerate point intervals are legal
v1). A disagreement between declaration and solid is a named
`RealizeError` value (errors are data) -- never a silent
preference for either side.

**The eventual producer.** hematite already has the language
surface: `.cavity(inlet=...)` derived void entities, computed at
L4 (`docs/hematite/02-language.md` sec. 6). Lowering will populate
`flow_paths` from cavity queries once the profile/Walk surface is
promoted (the scope note in `crates/regolith-ir/src/
feature_program.rs`). Until then, hand-authored `FeatureProgram`
fixtures declaring `flow_paths` are legitimate producers -- the
same AD-22 producer-role precedent that already covers
hand-authored realized records. Deferred with a reopen criterion
in hematite/07 sec. 2a (added this cycle).

Normative home: this entry + the amended
`regolith.realizer.mech.schema` docstrings when the change lands.
Machinery: WO-42 deliverable 4 (amended this cycle).

## D131. One wire shape for realized mech geometry (repairing the deliverable-1 drift)

**The problem (NO DUPLICATION tripwire, already sprung).** Two
shapes exist for the same content: `regolith_oblig::
RealizedGeometry` (WO-42 deliverable 1: `stages` list,
`WettedSegment` with `bend_count: u32`, a 3-value `RoughnessClass`
enum, point-valued scalars, per-STAGE wall) and the extract seam's
private `RealizedRecord` (WO-32 D2: selector-keyed `paths`,
segments with `bend {angle, radius}`, free-string roughness labels
validated against the 5-entry table, `[lo, hi]` interval bounds,
per-SEGMENT wall). Deliverable 3 hands realized-input bytes
straight to `extract_path`, which decodes the SEAM shape -- so a
realizer emitting deliverable-1 bytes would fail to decode. The
fixture-driven tests pass only because every fixture is
hand-authored seam-shape JSON.

**The decision.** The CONSUMER's shape wins; there is exactly one
wire shape and it lives in `regolith-oblig` (AD-5):

- `RealizedGeometry` is amended to: `feature_program_hash`,
  `step_content_hash`, `topology`, and selector-keyed `paths`
  whose segments carry the seam's field list verbatim (`role`,
  `flow_area [lo,hi]`, `length [lo,hi]`, optional `bend {angle,
  radius}`, `roughness_class: String`, `elevation_change [lo,hi]`,
  optional `wall {youngs_modulus, thickness, diameter}`). All
  bounds are `[lo, hi]` SI intervals (soundness: realized
  dimensions carry process capability; a v1 realizer may emit
  degenerate points).
- REMOVED, never shipped to any consumer, no migration:
  `RealizedStage`, `WettedSegment.bend_count`, the
  `RoughnessClass` enum, per-stage `WallData`, point-valued
  scalars. Per-stage structure IS the selector convention
  (`<stage_name>.wetted`, D130); WO-42 deliverable 1's "per-stage
  structure" phrasing is amended accordingly.
- `regolith-lower::extract` decodes the schemars type and DELETES
  its private `RealizedRecord`/`RawSegment` mirror in the same
  change. The in-record `snapshot_hash` dies with the mirror:
  D128 already pinned the citation -- extracted values cite the
  supplied IR's payload digest, which the caller (the
  realized-input channel) knows.
- Byte encoding on the channel/store follows the `FlownetPayload`
  precedent (AD-18's canonical encoder); the extract tests'
  hand-authored fixtures are re-keyed to the unified shape in the
  same change.
- One `SCHEMA_VERSION` bump for the whole amendment (coordinate
  the number at dispatch, per the WO-42 deliverable-1 precedent).

Normative home: this entry; `crates/regolith-oblig/src/
geometry.rs` module docs cite it when the change lands.
Machinery: WO-42 deliverable 1 (amendment note) + deliverable 4.
