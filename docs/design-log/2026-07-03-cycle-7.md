# Design cycle 7 -- 2026-07-03 (closing the judgment queue)

Mandate (user directive): resolve everything a less-capable agent
should not be trusted to resolve, so that what remains is mechanical
(implementation WOs, script generation) or the owner's taste
(naming). Each resolution below is the kind that needed either a
soundness argument, a cross-track vocabulary judgment, or a
schema-vs-reality test -- exactly the failure modes of a weaker

Numbering continues (F85.., D55..). Mantra order:
Unambiguous > Intent-based > User-friendly.

## A. Decisions

**D55. SOPEN-6 closed: the geom role kit** (substrate `10` sec. 3a).
Seven domain-neutral geometric role predicates (`point`, `axis`/
`plane`, `plate`, `region`, `hole_pattern`, `boss_pattern`,
`envelope`), each defined as a **declared-measures + T2-measurement
pair that both realizers can evaluate** -- that pairing IS the
domain-neutrality criterion, and it is why the kit is small: a
predicate neither realizer can measure has no business in a
cross-domain contract. Layouts reuse the shared constructors; kit
roles export derived datums (`.center`, `.pattern_center`, `.plane`,
`.normal`); any role may carry an `at=geom.point(...)` anchor (the
boundary-load `at=` reused). Versioned registry content under
coherence -- extended by publishing, never forked. The Kestrel
contract pack (F76's requirements list) now conforms.
- *Unambiguous:* one kit, closed v1 set, no per-pack invention.
- Danger avoided: a weaker agent would have blessed Kestrel's ad-hoc
  spellings as-is, without the T2-pair criterion that makes the set
  principled and small.

**D56. SOPEN-5 closed: logarithmic unit views** (substrate `02`
sec. 5a). Log units are *views* of linear quantities -- stored,
solved, and cached linear; parsed and printed in dB-family units.
Referenced units (dBm, dBW, dBuV) carry a reference dimension;
unreferenced ones (dB, dBi, dBc) view ratios. **Sum legality =
linear product legality**: after cancelling subtracted references
against added ones, at most one referenced term may remain --
validated by a grammar experiment (scratchpad `log_units.py`) that
accepts every link-budget shape (P_tx + G_ant - L_path -> power;
P_rx - P_sens -> ratio) and rejects the three classic bugs
(dBm + dBm = mW^2, dBm + dBuV, a lone negative reference) at L1.
Because the view is strictly monotone, interval corners commute
through it: corner discipline and margin math run in linear space,
untouched -- no second numeric domain exists. INV-17's mechanism
gains the reference check in the same change (ledger house rule).
The Kestrel link budget upgraded from `assume!` to a real dB claim;
the comms PA claim now reads `>= 26dBm`.
- Danger avoided: naive "dB is just a unit" makes `dBm + dBm`
  well-typed; naive "dB is special math" forks margin/corner
  machinery. The view rule gets both right with one L1 check.

**D57. EOPEN-18 closed: energy-harvest vocabulary** (elec `02`;
substrate `02` sec. 5 profile windows).
1. `supply:` is reserved for **definite** sources; a harvesting
   product declares the environmental resource as profile-structured
   boundary truth (`illumination: solar(..., profile=spectrum(ref))`).
2. `convert(<from> -> <to>)`: the verb schema takes conversion
   endpoints; the arrow is the flow arrow -- a conversion is a typed
   flow with an efficiency claim.
3. `store(q)` retains a **quantity** -- information or energy, one
   idea (retention), registered overload; schema params follow the
   quantity (bits: size/retention; energy: capacity/cycles/
   self-discharge).
4. **Profile windows**: `over=` in accumulation claims accepts
   durations, event windows, or hash-pinned periodic profiles;
   profiles integrate mode-weighted over one period, worst-case phase
   per corner discipline. This one bullet also serves duty-cycled
   fatigue and endurance claims -- it is quantity-core, not elec.

**D58. EOPEN-12 settled: the component record schema, proven on
three real parts** (`examples/registry/`). The G0 (atomic functions)
needs only the flat pin table. The SAMD21 broke it as predicted --
see F85 -- and the fix is the `function_modes:` table: per-mode
role->pad constraint rows, including joint-legality `combos:` where
role placements are not independent (SPI DOPO/DIPO). The RP2040
covers the remaining shapes: column *rules* instead of per-pin rows
(`spi.rx: gpio(n) where n mod 8 in {0, 4}`), a wildcard function
(PIO: `any(gpio0 .. gpio29)`) handled as an ordinary large domain,
and a mandatory companion (`flash: qspi_nor(...) required` -- the
missing-pull-up mechanism reused for boot flash). Pin-mux remains one
monomorphized matching problem over
function instance -> mode -> role -> pad -> pin. Transcription risk
is carried honestly: records are `tier=community` until the
datasheet-rev hash is verified, so a `trust: >= certified` claim
group refuses them -- correctly.

**D59. Claim subject `all` canonicalized; `all_parts` retired.**
Bare `all` means the FULL subject domain of the claim form -- parts
for `mech.mass`, consumers for `elec.power`, executors for
`info.utilization` -- the same one idea as the `.all` cardinality
intent (explicitly everything, domain given by context type). Was
watchlist F63; resolved now rather than at WO-13 because the
script-generation phase would otherwise mint hundreds of files
imitating two spellings. Corpus swept (six files); mech `04` gains
the row.

**D60. OPEN-7 settled for v1: no host language in design source.**
The generative constructs are the closed set (PatternOf, variants,
orbits + orbit connections, monomorphized domains, parameterized
`from_fn`/`from_table`). Rationale is the charter itself: source
must be statically checkable without executing anything -- a Turing
escape in design source breaks local verifiability, the reason the
languages exist. v2 may add a plugin API restricted to *registry
content* (harness models, rule packs, format readers), which is
already versioned, evidenced, and outside design source.

**D61. OPEN-12 closed** (per-stage plan obligations; setups are
internal plan structure) -- the long-standing leaning, exercised by
the gear reducer's mixed supplied+generated setups; nothing more was
ever going to arrive.

## B. Findings

**F85. Flat pin->function tables collapse on the SAMD21.** A SERCOM
is protocol-polymorphic and each protocol constrains roles to pads
(i2c: sda=PAD0/scl=PAD1 fixed; uart tx: PAD0|PAD2; spi: DOPO/DIPO
joint combos). The record schema therefore carries TWO tables: pins
declare pad capabilities; `function_modes:` declares per-mode
role->pad constraints (with `combos:` where placements are joint).
This was the exact failure EOPEN-12 was held open to find.

**F86. The one-reference rule is the whole of dB soundness.** All
three classic link-budget type errors reduce to reference counting
after cancellation; no other check is needed, and nothing downstream
of L1 changes. (Experiment: `log_units.py`.)

**F87. Profile windows generalize beyond energy.** The `over=`
profile spelling designed for orbit energy is verbatim the
duty-cycled fatigue window and the storage-endurance window --
placed in the quantity core (substrate `02` sec. 5), not in elec,
for that reason.
