# Design cycle 6 -- 2026-07-03 (the LARGE-project stress test)

Method (user directive): implement one complicated, genuinely large
project in target syntax and see where the specs fail. The subject is
**Kestrel** (`examples/cubesat/`), a 1U Earth-observation CubeSat:
ten files, both languages, a shared contract pack, four boards, a
machined+sheet structure, a deployable mechanism, an FPGA payload,
firmware, a flatsat target, and the first real project manifest. It
was chosen to be hostile on purpose: hard mass/energy budgets, launch
vibration masks, vacuum thermal, self-actuation (burn-wire deploy),
RF link margins, and an intent->workload->silicon chain that must
survive being split across files. Numbering continues (F74.., D47..).
Mantra order in force: Unambiguous > Intent-based > User-friendly.

Verdict up front: the substrate held. Nothing in the contract model,
ownership, claims/evidence, or the ladder had to bend. The failures
were (a) unstated resolution rules that only multi-file, multi-team
composition exposes, (b) vocabulary that existed in one track but was
needed by the other, and (c) two genuine domain gaps (log quantities,
energy-harvest boundaries) now ledgered as open questions rather than
papered over.

## A. Decisions (the example forced each one)

**D47. EOPEN-17 closed: host binding.** The payload
(`payload.loom`) is the second worked FPGA example the question was
waiting for, with a real multi-bank conflict: sub-LVDS camera lanes
need a 2.5V-Vccio bank while the OBC-facing SPI needs a 3.3V bank --
two DIFFERENT banks of one ECP5. The cycle-4 shape (per-bank
capability records, demand vectors, planner cause, `hosted_on` pin)
expressed it with nothing new; the single-part case even drops the
pin (planner-allocated, lockfile-caused). Settled in elec `05`.

**D48. Intent partition pins: inline `hosted_on <part>`.** [SETTLED]
elec `02` sec. 5 said "the human may sketch the partition" and gave
NO syntax -- unfindable until a project had five boards to partition
across. Spelling: `sense_att: sense(geom.orientation) hosted_on adcs:`
-- the existing lock-family word, one idea ("this derived content
lives on that part"), rung 2 semantics: checks unchanged, spurious
failure possible, silent pass impossible, lockfile-mirrored.
- *Unambiguous:* reuses a registered word for the identical idea
  rather than minting `on`/`via`/`assign` variants.

**D49. Budget `kind=` is pack-provided.** [SETTLED]
The mass budget -- THE budget of any satellite -- had no kind: the
docs enumerated tolerance/deflection (mech) and power/timing/noise/
error (elec) as if closed. Rule: `kind=` names a budget-math pack
(interval sum shape + share semantics), extensible exactly like
intent verbs; std adds `mass` and `energy`; members may span domains
(Kestrel's mass budget covers boards AND structure -- substrate
budgets never cared, only the doc's example list implied a limit).
Substrate `04` sec. 5.4.

**D51. Relative path imports resolve against the importing file's
directory**, and must stay inside the project root. [SETTLED]
Unstated until `eps.loom` reused `../elec/buck_converter.loom`.
Content is hash-pinned regardless (INV-22), so the rule is about
name resolution, not reproducibility. Substrate `11` sec. 9.

**D53. Orbit connections are substrate vocabulary.** [SETTLED]
`pairwise`/`flatten`/broadcast lived in the elec track only; the
frame (`structure.mill`) needed to zip four panels onto four rails
the moment a mech assembly contained part orbits (`4 x Rail`). Mech
connections carry full mating contracts, so the zip takes one:
`pairwise(a_set, b_set) by <Mating>` instantiates one mating per
element pair in shared layout order. Substrate `04` sec. 4; mech `04`
sec. H rows.

**D54. Artifact-typed caller params.** [LEANING]
The deadlock: `flight_fw` (the image) must be declared at intent
altitude (its `realizes: schedule(FlightCore)` names the computer,
which names the intents), but `load(fw)` is a construction step of
the OBC board -- an upward import would cycle. Resolution: `<...>`
params may be artifact-typed -- `board ObcPcb<fw: image>:` ...
`firmware = load(fw)`, bound at instantiation
(`obc: ObcPcb(fw=flight_fw)`), exactly the variant-choice-at-binding
shape. Leaning not settled: one use case; watch whether it wants
restrictions (image/part kinds only?) before Phase B. Substrate `04`
sec. 1.

## B. Findings

**F74 -> D51.** Path-import resolution root was unstated.

**F75. Config-domain references cross artifact boundaries with no
stated rule.** The EPS claims `forall op:` (op is the TOP system's
mode variable); the frame claims `during deploy = stowed` (deploy is
exposed by the antenna's hold-down connection). The `boundary.` rule
(mech `03` sec. 4 rule 6) was built for exactly this shape but only
covered boundary names. Extended: config-domain names in sub-artifact
claims resolve per enclosing context; no context supplying the name
is a constructive error naming the missing exposer.

**F76 -> SOPEN-6. Mixed-domain interface DECLARATION vocabulary does
not exist.** Every previous cross-domain contract (PodMount,
DriveCartridge) was imported from a never-shown pack. Kestrel's
`contracts.loom` is the first declared one, and substrate `10`'s rule
("foreign-domain roles in quantity-core vocabulary") turned out to be
a promissory note: the `geom` namespace has quantities but no role
predicates. The pack invents `geom.plate`, `geom.hole_pattern`,
`geom.boss_pattern`, `geom.point` by necessity. Escalated: the geom
role-predicate set needs a design pass before any real cross-domain
pack ships.

**F77/F78 -> EOPEN-18. Energy-harvest boundaries are awkward.**
`supply:` wants a source; a satellite has an intermittent source
(illumination as boundary truth) plus DESIGNED storage. Expressed
today as `illumination:` boundary truth + `convert(solar -> dc_bus)`
/ `store(energy(...))` intents, and orbit-averaged energy claims
lean on spectrum refs (`over=boundary.orbit.profile`) -- workable,
verifiable, but heavy, and the harvest-vs-consumption comparison
(`energy(harvest) >= 1.15 * energy(all_loads)`) deserves a designed
claim form. Opened as EOPEN-18 with the working spelling recorded.

**F79. The intent -> workload -> silicon chain resists file
splitting.** `realizes` names intents, so the `computer` gravitates
to the intent's file (kestrel.loom), and `bind` then reaches into
boards for instances (`cpu0 = obc.u_mcu`). This works -- part-
qualified references are ordinary queries -- but it means the
integration file owns the computer even when "the OBC team" would
naturally own it. Recorded as a collaboration-shape consequence, not
fixed: moving the computer down would need cross-file intent
references and reopen D48's altitude question. Watch in Phase B.
**F79b**: discrete-state roles (`release: {held, released}`) and
connection-exposed config vars coupling through `couples:` worked
unchanged for the burn-wire linkage -- positive result.

**F80 -> D53.** Part orbits (`4 x Rail`) made a mech assembly need
orbit connections the elec track already had.

**F81 -> SOPEN-5. No logarithmic quantities.** The RF link budget is
dB arithmetic or it is unreadable; Kestrel carries `link_margin >=
6dB` as an `assume!` and states PA output in linear watts with an
apology comment. Honest, release-gated, and clearly a domain gap:
dB/dBm as affine unit views with additive composition is a
quantity-core decision, not a claim-form hack. Blocks nothing in v1.

**F82. Composite artifacts implementing interfaces.** A card bay's
four posts come from four DIFFERENT rails: an assembly-level `impl`
whose role entities span parts. Previously only parts impl'd
interfaces (assemblies retro-contracted imports). Stated in substrate
`04` sec. 2: bindings are queries over the composed entity DB; the
permanent borrow lands on each contributing part. The bay binding
also leaned on `along()` patterns exporting a `deck` measure --
height along a rail is semantic content, the bus-bit argument; noted
for the query-predicate catalog (Phase A).

**F83 -> D54.** The image-loading deadlock.

**F84. Small dodges the corpus should remember** (each self-caught
while writing, worth keeping as agent-trap examples): a floating
`todo!` outside impl position is illegal (deleted); `10:30` as a
time-of-day literal fights the grammar (use `10.5h`); "per pass" is
not a unit (rates normalized to `/day`); a fault event must be
declared (`event anomaly: fault(any_part)`) before `within .. after`
can cite it; promise slots may be mode-qualified (`power: <= 2.8W
during op = downlink`) -- already anticipated by elec `07`'s
"consumption envelope per mode".

## C. What deliberately did NOT change

- No statechart: five modes + `forall op` + `during op = x` carried
  the whole mission profile (EOPEN-8's leaning survives its hardest
  test yet).
- No new deferral mechanism: the link budget, TID data, and radiator
  model land on `assume!`/`by test`/trust floors exactly as designed.
- The stack bus needed no panel/backplane special form: one harness
  interface, one card interface, one mating, four instances.
- `communicate` stayed boundary-only: every inter-board link in
  Kestrel is a derived flow realization; the only communicate intents
  face the ground station.
