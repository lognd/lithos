# dcad — Syntax Reference & Lowering Map (Working Draft)

> Purpose: one document showing (1) the canonical syntax of every construct,
> (2) the full lowering stack and what exists at each level, (3) a construct ×
> level matrix, and (4) an honest clunkiness audit with proposed fixes.
> This is the "feel of the whole system" document. Nothing here is final;
> audit items marked **[FIX-n]** are proposals awaiting your call.

---

## 0. The Mantras (and what enforces them)

| mantra | meaning | enforcement mechanism |
|---|---|---|
| **Unambiguous** | every reference, choice, and behavior has exactly one valid interpretation | query cardinality typing; ownership + borrows; orbit-checked `any`; branch pins in sketches; merge-sign rules; DOF ledger; monomorphization; banned `==`; banned inheritance |
| **Intent-based** | the human declares purpose and boundary truth; the system derives consequences | boundary-only inputs; `derived`/`allocated` contracts; goal-oriented free variables; requirements as claims; process-aware features; interfaces before parts |
| **User-friendly** | the cheapest tier catches each error, constructively, in the user's vocabulary | static-before-geometric-before-physical; Rust-style geometric diagnostics with fixes; margin-driven fidelity (no knobs); lockfile-visible defaults; `todo!`/`assume!` as honest state; preview/LSP/round-trip GUI as projections of text |

Supporting law — **the defaults test**: a default is permissible iff
conservative, local in effect, and lockfile-materialized. Otherwise it must be
a declaration.

## 1. The Unified Value-Source Grammar **[FIX-1: adopt everywhere]**

Audit finding promoted to the front because it simplifies everything below.
We currently have ~7 spellings for "who decides this number": `var(a,b)`,
`free`, `allocated`, `derived(...)`, `rated(x)`, `promised(>=x)`, `± allocated`.
These are genuinely different *mechanisms*, but they deserve one grammar.
Every numeric slot in the language (dimension, tolerance, load, stiffness,
spring rate, preload) takes exactly one of **five sources**:

```
<value-source> ::=
    <literal>                  # human knows it:        5mm, 25bar, 80kN/mm
  | in [lo, hi] [opt-dir]      # optimizer decides:     in [1mm,6mm] minimize
  | free                       # DFM minimum decides:   the legal cheapest
  | derived [(margin=k)]       # statics consequence:   computed, pinned
  | allocated [(policy)]       # budget-shared:         allocated cost_optimal
```

So: `radius = in [2mm, 8mm]` replaces `var(2mm,8mm)`; a load is
`radial: derived(margin=1.5)` or `radial: 5kN` (literal = old `rated`/
`explicit`); stiffness `radial: allocated` or `radial: >= 80kN/mm` (a literal
bound); tolerance `± allocated` unchanged. `promised`/`rated` vocabulary is
deleted — the *slot* (being in an interface) is what makes a value a promise.
Comparator literals (`>= x`, `<= x`, two-sided for flexures) are literals.

Every non-literal source resolves into `dcad.lock` with its resolving cause.
One grammar, five intents, taught once.

## 2. Construct Syntax Inventory

### 2.1 Quantity core (shared package)

```
namespace mech:
    quantity stress:       Pa,   tensor(2)
    quantity displacement: m,    vector
namespace thermo:
    quantity temperature:  K,    scalar

# interval values everywhere a scalar is legal:  [300K, 900K]
# zone-indexed fields (SEAM-4, from igniter P1):
#   thermal: zones(tip: [700,950]K, base: [300,400]K)   # boundaries = datums
```

### 2.2 Materials & pairs (stdlib database, hash-pinned)

```
material AISI_4140:
    E:  f(T) interval        # temperature-indexed (igniter P5)
    σ_y: f(T) interval per condition(annealed | quench_temper(...))
    ρ, ν, CTE: ...
    fatigue: sn_curve(ref=mil_hdbk_5j, R=-1, surface=per_finishing_stage)

pairs (steel, steel):
    μ_static: [0.5, 0.8] dry | [0.10, 0.16] greased
# user writes only:  material: AISI_4140   and   lubrication: greased
# overrides must cite evidence and stay intervals:
#   μ: [0.12, 0.14] by test(report_ref)
```

### 2.3 Process modules & capability

```
process cnc_mill(axes=3):
    capability:
        bore(d<20mm):        IT9,  Ra 3.2
        bore(d<20mm, ream):  IT7,  Ra 0.8
        face:                ±0.05mm, Ra 1.6        # finish column (igniter P4)
        position:            ±0.1mm
    dfm:
        tool_access(...), min_internal_radius(tool_d/2), ...
```

### 2.4 Profile (sketch) — topology/metric split

```
profile FlameTubeBore:
    walk:                          # topology + ALL discrete choices (branch pins)
        from inlet_plane
        line down                  # a
        arc tangent, bulge=right   # b
        line angled                # c
        arc tangent, bulge=left    # d
        line down                  # e
        close via axis
    constraints:                   # metrics: orderless, declarative
        a.length = 8mm
        c.angle = 30° from axis
        d.radius = in [1.5mm, 4mm]
        e.diameter = Ø6mm H9
        inlet_plane to throat_plane = 22mm ± allocated
    exports:
        throat_plane: datum
# rules: sketch DOF ledger must close (or declare free vars);
# unpinned solver branches are compile errors; curves are
# from_table(...) | from_fn(...), never freehand.
```

### 2.5 Part, stages, scopes, features, queries, datums

```
part IgniterBody:
    material: AISI_316
    stage barstock: process=saw_stock(Ø45mm)
    stage turned:   process=cnc_lathe, from=barstock
        then:
            envelope = Revolve(profile=OuterProfile)
            bore     = Revolve(profile=FlameTubeBore, internal)
        then seal_prep on envelope.front:            # labeled, region-guarded
            seal_face = Face(turn+polish)            # Ra from capability
    stage milled: process=cnc_mill(axes=4), from=turned
        then:
            ports = PatternOf<SwirlPort>(n=6, circular(Ø14mm),
                        frame=bore.axis.offset(z=18mm)
                                  .tangent_to(Ø14mm, elevation=8°))   # frame algebra (P2)
        then:
            plug_port = ThreadedHole(M18x1.5, on=envelope.side, ...)
            # C6 broken here; ports.any afterward = E0502, correctly

    datum throat_ref = bore.throat_plane            # immutable, borrow-exempt

    impl FlameSeat<Ø24> for self:
        seat = turned.seal_face                      # permanent borrow
```

```
# queries (canonical form is method-chain; bracket sugar removed [FIX-3]):
body.edges.where(adjacent=top, dir=Z)
edges.at_intersection(slot, base.top)        # explicit cross-owner join
pattern.instances.nearest(throat_ref)
pattern.instances.any                        # orbit-checked
q.all / q.only                               # cardinality intents
# cavity entities (igniter P3):
body.cavity(inlet=bore.inlet).wetted_faces
```

Scope semantics: all features in a `then:` block read the scope-entry
snapshot; exports visible next block; same-sign overlaps merge (ownership
declared only if queried), mixed-sign overlaps are hard errors; borrows live
to end of stage; interface bindings borrow for part lifetime.

### 2.6 Interface

```
interface BearingSeat<d: length, depth: length>:
    frame:
        origin: bore.axis ∩ shoulder.plane
        z: bore.axis                          # x,y free ⇒ C∞ declared
    roles:
        bore:     cylindrical(d=d, depth >= depth), internal
        shoulder: planar, normal=+z, annular(od >= d+8mm)
    tolerances:
        bore:     tol_class(H7), cylindricity(0.008mm)
        shoulder: perpendicularity(0.01mm, to=z), Ra 0.8
    loads:
        radial: derived(margin=1.5)           # value-source grammar
        axial:  derived(margin=1.5), structure: inherit
    stiffness:
        radial: allocated
    thermal:
        zones(seat: derived)                  # zone-valued (SEAM-4)
```

Composition: bundling (`PilotedFlange = Pilot + PatternOf<...>`) and
`refines` only. Evidence clauses on promises:
`by analysis (default) | by catalog(ref) | by test(ref)`.

### 2.8 Mating

```
mating BoltedFlange<n: int, bolt: BoltSpec, circle: length>:
    between: a: FlangeFace, b: FlangeFace     # NAMED sides [FIX-2]
    align:   a.frame = b.frame (z anti-parallel, contact)
    dof:     removed=[all]
    preload: torque: 25N·m → P0, scatter=[0.75,1.25]
    model:   joint_diagram(...)               # harness node ref, validity-domained
    state_requirements:
        no_separation: ..., SF=1.5
        no_slip:       μ_min · ..., SF=1.3    # corner discipline
    generates:
        → a, b:  clamp_pressure(P0_max·n, over=contact_annulus)
        → bolts: static(...), alternating(Φ·F_alt)
    capability:
        shear: μ_min·P0_min·n

mating GearMesh<ratio>:
    couples: θ_a = -ratio·θ_b                 # kinematic coupling
mating Compression(seat_a, seat_b):
    k: allocated;  free_length: ...;  preload_at: θ=0°
```

Escalations are declared: `stiffness=measured`, `model=fea_contact`
(build-graph edges, system-test tier).

### 2.9 Assembly

```
assembly TorchIgniter:
    parts:
        boss:  import("chamber_boss_rev3.step") verify-only
        body:  IgniterBody
        plug:  vendor(ngk_me8)
    boundary:                                  # the ONLY human numbers
        chamber_pressure: [0, 25bar], structure: spectrum(cycles_ref)
        chamber_wall_temp: zone(boss.inner): [300K, 900K]
        design_life: 5000 cycles
    connect:
        mount: BoltedFlange<4, M6, Ø42>(a=flange.base, b=boss.pad)
        pivot: Revolute(body.boss, nozzle.hub) exposing θ: angle in [0°, 95°]
        rear:  BearingMount(...) redundant(axial)    # declared indeterminacy
    environment:
        temperature: [-20°C, 95°C]
    require Seal:
        leak(mount.seal) < 1e-3 scc/s @ 25bar
    require Function:
        forall θ: closing_torque(θ) > 0.05 N·m
        equilibrium(θ=0°): stable
    require Fatigue:
        life(all_parts) >= design_life, scatter_factor=4
    budget mesh_alignment: kind=deflection
        require: misalignment(...) < 0.05mm
        members: [housing, shaft];  allocate: cost_optimal
        pins: shaft.seat_runout: locked(±0.015)
```

Config variables are namespaced by connection (`pivot.θ`). Assembly compute
uses promises only; stiffness network is the default solver, rigid statics
its determinate fast path; declared `redundant` routes through compliance and
couples tolerance intervals into misfit loads.

### 2.10 Claims & deferral

```
require Structural:
    mech.max(stress.von_mises) < material.σ_y(T_local) / 2.5
    @hint(regime=linear_elastic)              # droppable
ignition: assume!(torch_flux > 1.2MW/m², basis="hotfire TR-114")
```

`todo!` defers impl proof; `assume!` defers claim evidence; both ledgered,
both refused by `--release` minus per-item waivers.

## 3. The Lowering Stack

```
L0  SOURCE        .dcad text + stdlib (materials, processes, matings) + imports
 │   parse, format-normalize
L1  TYPED AST     names resolved; quantity/unit checking; value-source &
 │                tolerance grammar; ==-ban; profile walks parsed
 │   lower contracts
L2  CONTRACT IR   ← "contracts live here" — GEOMETRY-FREE
 │     graph of: frames, interfaces (roles/promise slots), matings (align,
 │     dof, couples, generates-as-model-refs), boundary conditions, budgets,
 │     claims, config space
 │     checks/solves: DOF & Grübler ledger · capability/fit table lookups ·
 │     rigid statics + promise stiffness network (incl. redundancy/misfit) ·
 │     derived-contract authoring (envelope over config & environment
 │     corners) · budget arithmetic · interface-first verification
 │   per-part lowering
L3  FEATURE IR    stages, scope DAG, features as predicted entity-DB deltas
 │     checks: query typing/cardinality · ownership/borrows/merge-signs ·
 │     monomorphization sweep · symmetry groups/orbits · sketch DOF ledgers &
 │     branch-pin completeness · topology-boundary domains · DFM eager
 │     propagation (resolves free) · T1 conformance
 │   kernel evaluation (only now does OCCT run)
L4  GEOMETRY IR   B-rep per stage; measured entity DB (imports enter HERE,
 │     skipping L3) · post-geometry prediction verification · T2 conformance ·
 │     mass properties · meshes · cavity derivation
 │   emit proof work
L5  OBLIGATIONS   ← "proofs live here" — self-contained, serializable
 │     from L2 (assembly/connection/state claims) and L4 (part claims, T3)
 │     harness: margin-driven model selection in validity domains →
 │     EVIDENCE {discharged | violated | indeterminate} → cache
 │     orchestrator: lazy loop over (in-domain ∩ topology-preserving) space,
 │     adjoint-guided; lockfile authorship
L6  BACKENDS      STEP/AP242+PMI (allocated tolerances ON the model),
                  drawings, G-code/toolpaths, BOM (vendor refs), ledger report
```

Two structural payoffs visible only at this altitude: **interface-first =
running the pipeline L0→L2 only** (a full system architecture verifies with
zero parts); **verify-only import = entering at L4** (zero feature machinery
needed) — the adoption wedge is literally a pipeline shortcut.

## 5. Clunkiness Audit

Each item: the smell, why it matters, the proposed fix.

**[FIX-1] Value-source babel.** Seven spellings for "compiler decides."
*Fix:* §1 grammar, five sources, used in every numeric slot. Deletes
`var()`, `rated()`, `promised()` as vocabulary. **Biggest single UX win.**

**[FIX-2] Positional mating arguments.** `BoltedFlange(x, y)` is ambiguous
when both sides are the same interface type (flange-to-flange) and silently
order-dependent otherwise. *Fix:* `between:` declares named sides
(`a:`, `b:`); use sites must name them. Symmetric matings may declare
`between: symmetric(FlangeFace)` to waive naming.

**[FIX-3] Two query syntaxes.** `.where(...)` chains vs `[...]` sugar — two
ways to write everything is a review tax and an LLM-confuser. *Fix:* method
chain is canonical; bracket sugar removed from the language (formatter could
have normalized it, but one form is friendlier than a normalize rule).

**[FIX-4] `with` blocks are vestigial.** Early design had
`with body.top as f:` for scoped selection; HDL scopes superseded the
execution半semantics but the region-guard idea was good. *Fix:* delete
`with`; scopes gain optional labels and region guards:
`then seal_prep on envelope.front:` — label feeds diagnostics ("in scope
'seal_prep'"), guard makes out-of-region selection a static error. One
construct, both jobs.

**[FIX-5] First-scope keyword asymmetry.** `scope:` then `then:` reads as
two constructs. *Fix:* every block is `then:` (a stage implicitly opens at
its process datum state). Asymmetry deleted.

**[FIX-6] Stage-path verbosity.** `machined.main_bore.wall` everywhere gets
heavy. *Fix:* within a stage, bare names; cross-stage references require the
stage qualifier; impl blocks (which sit outside stages) always qualify.
Matches Rust module-path intuition exactly.

**[FIX-7] Config-variable namespace.** `exposing θ` floated free; collisions
inevitable. *Fix:* config vars are fields of their connection
(`pivot.θ`), `forall` quantifies over named domains
(`forall pivot.θ:`). Already reflected above.

**[FIX-8] Anonymous claims.** `require Structural:` holds bare expressions;
evidence ledgers and waivers then lack stable handles. *Fix:* claims may take
names (`max_stress: mech.max(...) < ...`); the ledger, waivers
(`--waive Structural.max_stress`), and lockfile cite them. Unnamed claims get
derived handles but the linter nudges naming in `require` blocks > 1 claim.

**[FIX-9] Unicode operators.** `∩`, `Ø`, `±`, `°`, `μ` read beautifully and
type terribly. *Fix:* ASCII canonical forms (`&`, `dia`, `+-`, `deg`, `mu_`),
unicode accepted and pretty-printed by the formatter. Non-negotiable for
adoption; identical to how Julia/Lean handle it.

**[FIX-10] `verify-only` vs stage-import asymmetry.** Rung 1 uses a part-level
modifier; rung 2 uses `stage imported:`. Two spellings of "geometry enters at
L4." *Fix:* one form — every import is a stage
(`stage src: import(path) [sealed]`) where `sealed` = no later stage may
modify (rung 1). `verify-only` deleted; the ladder becomes purely "how many
stages follow the import stage."

**Watchlist (not yet actionable):** walk-grammar EBNF (Phase A); zone syntax
(SEAM-4) must not freeze obligation schema before it lands; `variant` design
(OPEN-1) now interacts with FIX-1 (`in [..]` over discrete sets?); budget
`kind=` may want a third kind (thermal) — check during zone design.

## 6. Coherence Check Against the Mantras

- **Unambiguous:** FIX-2/3/5/7/10 each remove a second way to say one thing;
  the matrix shows every construct has exactly one home level for each
  semantic job (contracts L2, ownership L3, measurement L4, proof L5).
- **Intent-based:** §1 grammar makes "who decides and why" the literal
  syntax of every number; boundary block remains the only place humans
  assert physical truth; L2-before-L3 ordering *is* intent-before-geometry.
- **User-friendly:** the stack's check placement = cheapest-tier-wins;
  FIX-4/6/8/9 are pure ergonomics; interface-first and import-at-L4 give the
  two on-ramps (greenfield top-down, brownfield verify-first) without special
  modes.

**Where I still feel residual friction, unresolved:** (a) the `generates →`
arrow blocks inside matings are powerful but dense — possibly the one place a
stdlib author's syntax leaks into end-user reading; may want a rendered
"connection datasheet" view rather than syntax change. (b) `then:` blocks
with one feature each (very sequential parts) will feel like boilerplate;
consider allowing a bare feature statement to imply its own scope. (c) The
boundary between profile `constraints:` and part-level `constraints` on
features uses one word for two scopes — likely fine, watch in Phase A.
