# dcad Spec 0.3 — The Vocabulary

> Every block and keyword in the language: where it is legal, what it means,
> and what it lowers to. One construct per idea; overloads permitted only
> where the underlying idea is **fundamentally identical** and context can
> never make the reading ambiguous (each such case is justified in §5).
> Changes from prior drafts are listed in §2 and marked ★ in the tables.

---

## 1. Vocabulary Principles

1. **One word, one idea.** A keyword may appear in multiple positions only if
   it means the *same thing* in all of them (§5 registry). Near-collisions in
   spelling (`fit`/`fix`) are treated as bugs even when meanings differ.
2. **Blocks declare, sources resolve, claims prove.** Every construct is one
   of: a *declaration* (structure), a *value source* (who decides a number),
   a *claim* (what must be true), or a *selector* (which entities). No
   construct straddles categories.
3. **Human words at human boundaries.** Anything the user must type often is
   short and reads aloud (`then`, `hold`, `free`); anything rare and
   load-bearing may be long (`scatter_factor`).
4. **ASCII canonical, unicode pretty.** `&`, `dia`, `+-`, `deg`, `mu_` are
   canonical; the formatter renders `∩ Ø ± ° μ`.

## 2. Vocabulary Changes in 0.3

| id | change | reason |
|---|---|---|
| ★V1 | `environment:` **deleted**; ambient conditions move into `boundary:` | two homes for "external truth declared by the human" was a split brain — wall temperature lived in `boundary`, ambient in `environment`, same kind of fact |
| ★V2 | `state_requirements:` **deleted**; matings use `require <Group>:` like everything else | state requirements *are* claims; one claim construct everywhere |
| ★V3 | setup `fix:` → **`hold:`** | `fix`/`fit` one-letter collision is a typo hazard; `hold` is the trade term (workholding) anyway |
| ★V4 | safety multiplier is **`sf=`** everywhere (`derived(sf=1.5)`, `... > 0, sf=1.5`); **`margin` now means only** the evidence value-to-limit distance | we had `margin=1.5` (multiplier) and evidence `margin: 11.1 kN/mm` (distance) — same word, two meanings, and the multiplier usage was wrong engineering vocabulary |
| ★V5 | material-class member list `requires` → **`properties:`** | `requires`/`require` near-collision; a class property schema is not a physical claim |
| ★V6 | signature `domain_decl` → **`domain:`**; DFM rule three-way output adopts the same field | one word for validity domains across signatures, impls, and rules |
| ★V7 | mating `generates:` → **`effects:`** with `model<sig>(...)` references and `applies ->` routing | "generates" implied the mating computes; it only *declares* — physics lives behind signatures |
| ★V8 | guarded-scope composition rule: inside `then L on R:`, feature placement `on=` **defaults to R** | removes double-typing of the region; guard and placement are the same spatial context |

Previously retired (0.2 → syntax audit), listed for completeness:
`var()`, `rated()`, `promised()`, `pairs`, `with`, `scope:`, `verify-only`,
bracket query sugar, `==` on continuous quantities, positional mating sides.

## 3. The Vocabulary

### A. Shared registries (neither compiler nor harness owns these)

| keyword | position | purpose |
|---|---|---|
| `namespace` | top-level (quantity core) | groups physical quantities (`mech`, `thermo`, `geom`) |
| `quantity` | in `namespace` | declares a typed physical quantity: unit, tensor rank; interval- and zone-valuable |
| `zones(...)` | quantity values | zone-indexed field value; zone boundaries are datums |
| `class` | top-level (materials) | material class hierarchy node (`steel: metal`) |
| `properties:` ★V5 | in `class` | the property schema members of this class must provide |
| `material` | top-level | a concrete material: `f(T)` interval properties, `condition(...)` variants, fatigue data with provenance |
| `contact { A, B }` | top-level | unordered-pair property record over materials or classes (friction, conductance, wear); specificity-resolved |
| `process` | top-level | a manufacturing process module: `capability:` table (tolerance grades, Ra, refixture error) and `dfm:` rule pack |
| `capability:` | in `process`, in `mating` | provider envelope checked as demand <= capability (§5 justified overload) |
| `signature` | top-level | a physics model contract: `inputs:`, `outputs:` (quantity-core typed), `domain:` — the linker symbol between Parts 1 and 2 |
| `impl <sig> by <name>` | harness packs | a model implementing a signature: cost, error model, `domain:` |

### B. Sketch layer

| keyword | position | purpose |
|---|---|---|
| `profile` | top-level / in part | a 2D constraint system; a *value* (borrow-exempt), not topology |
| `walk:` | in `profile` | the topology serialization: entity sequence + **all** discrete branch pins |
| `from <datum>` | first walk statement | the walk's start anchor (§5: same idea as stage `from=`) |
| `line`, `arc` | in `walk` | entities; direction words (`down`, `angled`) are uniqueness-checked hints, never approximations |
| `tangent`, `perpendicular` | arc/line joins | join-condition pins |
| `bulge=left/right` | on `arc` | branch pin, defined chord-wise in the +normal view |
| `close [via axis]` | last walk statement | closes the loop; `via axis` marks a revolve centerline closure |
| `hole <name>:` | in `profile` | a named inner loop (one nesting level only) |
| `regions:` | in `profile` | named area expressions (`interior - holes`, `between(...)`) consumable by features |
| `constraints:` | in `profile` | the metric layer: orderless dimensional/geometric constraints |
| `exports:` | in `profile` | datums and named sub-entities the profile publishes |
| `from_table()`, `from_fn()` | curve sources | data- or math-defined curves; freehand splines do not exist |

### C. Part layer

| keyword | position | purpose |
|---|---|---|
| `part` | top-level | a manufactured component: material + stage pipeline + impls + claims |
| `material:` | in `part` | binds a registry material (§5: same idea as the declaration) |
| `stage` | in `part` | one process step in the pipeline; `process=`, `from=<stage>`, or `import(path) [sealed]` |
| `sealed` | on import stage | no later stage may modify this geometry (rung-1 verify-only) |
| `setup` | in machining stages | a workholding state: `hold:`, datum letters, ordered; omitted setups are planner-`allocated` |
| `hold:` ★V3 | in `setup` | the gripped/fixtured geometry for this setup |
| `flip about <axis>` | in `setup` | re-fixturing transform; injects refixture tolerance from capability |
| `then [label] [on <region>]:` | in stage/setup | concurrent scope: all features read the entry snapshot; commits at block end; guard restricts selection and defaults placement ★V8 |
| `seq:` | in stage/setup | sequential sugar: each statement is its own commit; lint nudges independent features back to `then:` |
| `datum` | in part; letters in `setup` | immutable reference geometry; borrow-exempt; setup datum letters bind GD&T frames to fixturing |
| `impl <Interface> for <Part>` | in part | binds roles to geometry; permanent borrow; `= todo!` defers proof, never the contract |
| `merge(a before b / a over b)` | scope commit | explicit ordering/ownership for overlapping same-scope features |
| `rebind()` | feature arg | re-evaluates a borrow after an intervening modifier |
| `sequence: a before b` | in stage | pins manufacturing order otherwise `allocated` by the CAM planner |

### D. Contract layer

| keyword | position | purpose |
|---|---|---|
| `interface` | top-level | a contract: `frame:`, `roles:`, `tolerances:`, promise slots (`loads:`, `stiffness:`, `thermal:`) |
| `frame:` | in `interface` | the datum reference frame all contract content is expressed in; undeclared axes declare symmetry |
| `roles:` | in `interface` | named geometric slots with predicates (`cylindrical(d=d)`, `internal`) |
| `tolerances:` | in `interface` | GD&T demands on roles relative to the frame, incl. `Ra` |
| `loads:` / `stiffness:` / `thermal:` | in `interface` | promise slots; values use the source grammar (literal, `derived(sf=)`, `allocated`) |
| `structure:` | on loads | time structure: `static` (default) / `alternating(R=, f=)` / `spectrum(ref)`; propagates through derivation |
| `refines` | interface header | narrows an existing contract; one-way substitutable; the only inheritance-like mechanism |
| `mating` | top-level | a connection contract between named sides: `between: a: X, b: Y` |
| `align:` | in `mating` | frame-coincidence rules; the **only** positioning mechanism in assemblies |
| `fit:` | in `mating` / connection | ISO fit designation; expands to interval deviations; capability-checked |
| `dof:` | in `mating` | removed/free rigid-body freedoms; feeds the assembly ledger |
| `couples:` | in `mating` | kinematic constraint between config variables (gear ratio, cam law) |
| `preload:` | in `mating` | internal load state with method scatter interval |
| `effects:` ★V7 | in `mating` | declared physics: `model<signature>(inputs)` with `applies -> role` load routing |
| `capability:` | in `mating` | what the connection can transmit (computed at worst-case corners) |
| `require <Group>:` ★V2 | in `mating` | connection-state claims (no-slip, no-separation), written against signature outputs |

### E. Assembly layer

| keyword | position | purpose |
|---|---|---|
| `assembly` | top-level | the system: parts, connections, boundary truth, claims, budgets |
| `parts:` | in `assembly` | part bindings: native, `import(...)`, `vendor(...)` |
| `vendor()` | parts entry | an interface bundle with catalog evidence; internals never modeled |
| `boundary:` ★V1 | in `assembly` | **the only human-asserted physical truth**: applied loads, pressures, ambient and zone temperatures, design life, spectra |
| `connect:` | in `assembly` | named connection instances: `name: Mating(a=..., b=...)` |
| `exposing` | on connection | publishes a config variable: `exposing theta: angle in [0deg, 95deg]`; namespaced as `name.theta` |
| `redundant(<dof>)` | on connection | declares intentional static indeterminacy; routes the load path through the compliance solve and couples tolerance misfit into loads |
| `lubrication:` | on connection | contact state selecting fields within the resolved `contact` record |
| `budget` | in `assembly` | a named cross-part allocation scope: `kind=tolerance/deflection`, `members:`, `allocate:`, `pins:` |
| `members:` | in `budget` | the explicit blast radius |
| `allocate:` | in `budget` | allocation policy (`cost_optimal`, `worst_case`/`rss` math) |
| `pins:` / `locked()` | in `budget` | human-fixed values within an allocated set (§5: one "pin" idea) |

### F. Claims & evidence

| keyword | position | purpose |
|---|---|---|
| `require <Group>:` | part, assembly, mating | a named group of claims; claims may be individually named for ledgers and waivers |
| `forall <cfg> [in <domain>]:` | claim prefix | quantifies a claim over a configuration domain |
| `equilibrium(...): stable` | claim form | potential-energy stability claim |
| `sf=` ★V4 | in claims, `derived()` | multiplicative safety factor |
| `scatter_factor=` | fatigue claims | life-scatter divisor (kept long: rare and load-bearing) |
| `@hint(...)` | on claims | droppable verification hints (symmetry, regime) |
| `assume!(expr, basis=)` | claim position | accepts a claim without evidence; ledgered; release-gated |
| `todo!` | impl position | defers conformance proof; contract stays concrete; ledgered; release-gated |
| `by analysis / catalog(ref) / test(ref)` | on promises, overrides | evidence provenance (§5: one "by" idea) |

### G. Value sources & domains (every numeric slot, everywhere)

| form | meaning |
|---|---|
| literal (`5mm`, `25bar`, `>= 80kN/mm`) | the human knows it (comparators are one-sided literals; flexures use two-sided) |
| `in [lo, hi] [minimize/maximize]` | bounded freedom; optimizer decides; integer domains monomorphize |
| `free` | the DFM minimum-legal value decides |
| `derived [(sf=k)]` | a consequence of assembly statics over config/boundary corners; pinned to lockfile; re-derived only by explicit `dcad update --contracts` |
| `allocated [(policy)]` | a share of a declared budget or planner output (tolerances, stiffness, spring rates, setup plans, op sequences) |

### H. Queries & selectors

| form | purpose |
|---|---|
| `.where(pred, ...)` | filter by predicates (`adjacent=`, `dir=`, `from_feature=`, geometric measures) |
| `.all` / `.only` / `.any` | cardinality intents: explicit-everything / exactly-one / orbit-checked-representative |
| `.nearest(datum)` etc. | semantic instance addressing (never positional index) |
| `at_intersection(a, b)`, `&` | explicit cross-ownership joins |
| `.cavity(inlet=...)` | derived void entity; `wetted_faces`, `volume`, `min_section` |
| `.as_datum()` | captures reference geometry out of the borrow system |

### I. Coherence & override (shared by all three trait mechanisms)

| keyword | purpose |
|---|---|
| `override <record> by <evidence>` | shadows a registry record at the same key; evidence clause mandatory |
| `use { A, B }` / `use <impl>` | pins resolution when specificity or impl choice is ambiguous (`model=<impl>` on connections is this same pin) |

The three trait mechanisms — material classes + `contact`, interface
`refines`, signatures + harness impls — share one rulebook: canonical
(unordered) keys, unique-most-specific-or-error, explicit override with
evidence, lockfile provenance.

### J. Build & lifecycle words (CLI-facing, not source)

`check` / `build` / `optimize` / `--release` / `--waive <Group.claim>` /
`update --contracts` / lockfile causes (`dfm`, `budget:<name>`, `topology`,
`obligation:<id>`, `planner`).

## 5. Justified Overloads (the "fundamentally same" registry)

| word | positions | the one idea | ambiguity risk |
|---|---|---|---|
| `require` | part / assembly / mating | a claim needing evidence | none — identical semantics, identical lowering (L5 obligation) |
| `impl` | interface impls / signature impls | provide an implementation of a declared contract | none — different headers (`for Part` vs `by name`) |
| `capability` | process / mating | provider envelope, always checked demand <= capability | none — different quantity domains, same operator |
| `from` | stage `from=` / walk `from <datum>` | "begin from this basis" | low — different positions; both are start anchors |
| `on` | scope guard / feature placement | spatial context of an operation | none after ★V8 — guard *is* the default placement |
| `by` | evidence clauses / overrides / signature impls | provenance attribution | none |
| `as` | impl aliases / `as_datum()` | naming/aliasing | none |
| `in` | value domains / `forall` domains | domain membership | none — identical semantics |
| `model` | `model<sig>()` refs / `model=<impl>` pins | reference into the signature/impl system | none — the pin selects among impls of the ref |
| `pin` family (`pins:`, `locked`, `use`, `sequence:`) | budgets / coherence / planner | human fixes an otherwise-allocated resolution | none — one mental model, lockfile-visible |
| `constraints:` | profile only | (no longer overloaded — part-level constraint blocks do not exist; parts constrain via value sources, tolerances, and claims) | — |
| `datum` | part capture / setup letters | immutable reference geometry; setup letters *bind* datums to fixturing for GD&T frames | low — one concept, two binding sites |

## 6. Open Vocabulary (reserved, undesigned)

`variant` (OPEN-1: discrete design alternatives; must compose with the value-
source grammar) · workholding nouns for `hold:` (vise/chuck/fixture-plate
semantics) · zone-declaration syntax beyond `zones()` (SEAM-4) · time-domain
claim words (OPEN-9: `transient`, `impact`) · `joint`/mechanism sugar over
`exposing` · drawing/annotation vocabulary (L6).
